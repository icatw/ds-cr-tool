# GPT-CodeReview 需求规格说明书

## 1. 项目目标

开发一个简单的代码评审工具，通过获取Git差异内容并调用DeepSeek API进行智能评审，提供代码质量反馈。

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 Git差异获取
- 获取最新代码改动
- 获取指定提交的改动
- 获取提交范围的改动

#### 2.1.2 AI评审
- 调用DeepSeek API进行代码评审
- 生成评审意见和建议

#### 2.1.3 评审报告
- 格式化输出评审结果
- 突出显示关键问题

### 2.2 Git集成

#### 2.2.1 命令行工具
- diff命令：评审最新改动
- review命令：评审指定提交

#### 2.2.2 Git Hooks
- pre-commit：提交前评审
- pre-push：推送前评审

## 3. 系统架构

### 3.1 核心模块

```
┌─────────────┐
│  Git模块    │ 获取代码差异
└─────────────┘
      │
      ▼
┌─────────────┐
│  AI模块     │ 调用DeepSeek API
└─────────────┘
      │
      ▼
┌─────────────┐
│  报告模块   │ 格式化输出
└─────────────┘
```

### 3.2 模块说明

#### 3.2.1 Git模块 (pkg/git)
- 封装Git命令调用
- 解析差异内容
- 管理Git Hooks

#### 3.2.2 AI模块 (pkg/model)
- DeepSeek API客户端
- 评审提示词管理
- 结果解析处理

#### 3.2.3 报告模块 (pkg/review)
- 评审结果格式化
- 问题分类展示
- 建议内容整理
